# OpenVisionLab Labeling Studio 사용 가이드

이 문서는 처음 사용하는 작업자가 이미지 폴더를 연 뒤 객체탐지, 세그멘테이션 또는 이상탐지 라벨을 만들고 학습 모델을 검토하는 순서를 설명합니다.

세 작업은 같은 레시피 구조를 사용하지만 정답의 형태가 다릅니다.

| 목적 | 한 이미지의 정답 | 대표 도구 |
| --- | --- | --- |
| 객체탐지 | 클래스와 사각형 위치 | 박스, 선택, 이동, 삭제 |
| 세그멘테이션 | 클래스와 픽셀 영역/외곽선 | 폴리곤, 브러시, 지우개 |
| 이상탐지 | 이미지 전체의 정상(OK)/이상(NG) | OK/NG 판정 버튼 |

추가 화면에 번호와 화살표가 필요한 경우 [화면 캡처 튜토리얼](labeling-workbench-tutorial.html)을 함께 보세요.

## 전체 작업 순서

```text
1 데이터셋 → 2 라벨링 → 4 학습/모델 → 후보 검증·비교 → 현재 검사 모델로 채택 → 3 AI 후보 검토(선택)
```

`3 AI 후보`는 직접 라벨링에 필수인 단계가 아닙니다. 이미 학습된 현재 검사 모델로 자동 라벨 후보를 만들 때만 사용합니다.

![라벨링 워크벤치 전체 화면](images/workflows-20260722/object-detection-labeling-1920x1080.png)

예시 화면은 결함이 포함된 NG 샘플을 사용합니다. 우상단에 눈으로 확인되는 대각선 `scratch_crack`과 저장 라벨이 같은 위치에 있는지 먼저 확인하세요.

## 1. 데이터셋 만들기

1. 상단의 `1 데이터셋`을 선택합니다.
2. `새로 만들기`에서 데이터셋 이름과 목적을 정합니다.
3. `이미지 폴더`에서 원본 이미지가 들어 있는 루트 폴더를 선택합니다.
4. `저장 폴더`에서 레시피, 정답 라벨과 학습 산출물을 기록할 별도 위치를 선택합니다.
5. 요약에 표시된 목적, 이미지 수, 클래스와 저장 위치를 확인합니다.

이미지 루트 아래의 하위 폴더도 한 이미지 큐에 포함됩니다. 예를 들어 `images/OK/...`와 `images/NG/...`가 있으면 `images` 하나만 선택합니다. 폴더를 각각 따로 불러올 필요가 없습니다.

저장 폴더가 같으면 기존 라벨을 다시 읽습니다. 새 실험을 만들 때는 이미지 폴더 이름만 바꾸지 말고 새 저장 폴더를 사용하세요.

## 2. 클래스 정하기

객체탐지와 세그멘테이션은 라벨을 만들기 전에 실제 검사 기준에 맞는 클래스를 등록합니다.

예:

```text
scratch
contamination
edge_chip
```

클래스 순서는 YOLO class index와 모델 출력 매핑에 사용됩니다. 학습을 시작한 뒤 이름이나 순서를 바꾸면 기존 정답과 모델 결과가 다른 의미가 될 수 있으므로 먼저 데이터셋 호환성을 확인합니다.

이상탐지의 기본 정답은 `normal`과 `abnormal` 두 상태이므로 결함 종류 클래스를 별도로 만들지 않아도 됩니다.

## 3. 객체탐지 라벨링

![객체탐지 사용 화면](images/workflows-20260722/object-detection-labeling-1920x1080.png)

이 화면의 빨간 박스는 임의 위치가 아니라 눈에 보이는 대각선 `scratch_crack` 정답 영역입니다. 박스는 결함 전체를 포함하되 주변 정상 배경을 과도하게 포함하지 않습니다.

1. 데이터셋 목적을 `객체 탐지`로 선택합니다.
2. `2 라벨링`으로 이동합니다.
3. 캔버스 왼쪽 도구 레일에서 `박스`를 선택합니다.
4. 대상이 모두 들어가도록 드래그하되 불필요한 배경은 줄입니다.
5. 왼쪽 `선택 라벨 수정`에서 클래스를 확인하고 필요하면 `적용`합니다.
6. 상단 `라벨 저장`을 누릅니다.
7. 오른쪽 이미지 큐의 저장 상태가 `저장됨`인지 확인합니다.
8. 대상이 없는 이미지는 `객체 없음`으로 명시해 미작업 이미지와 구분합니다.

모든 이미지에서 박스 기준을 동일하게 유지하세요. 일부 이미지만 외곽 여백이나 포함 범위가 다르면 모델 비교 전에 라벨 편차가 먼저 영향을 줍니다.

저장 후 필요하면 `미검토`, `수정 필요`, `검수 완료`로 품질 검수 상태를 기록합니다. 라벨을 다시 편집하면 검수 상태도 다시 확인해야 합니다.

## 4. 세그멘테이션 라벨링

![세그멘테이션 사용 화면](images/workflows-20260722/segmentation-labeling-1920x1080.png)

이 화면은 위 객체탐지 예시와 같은 `scratch_crack`을 10점 폴리곤으로 다시 연 결과입니다. 굵은 사각형 대신 실제 스크래치 경계를 따라가므로 위치뿐 아니라 결함 형상도 학습 정답에 남습니다.

1. 데이터셋 목적을 `세그멘테이션`으로 선택합니다.
2. `2 라벨링`으로 이동하고 클래스를 선택합니다.
3. 윤곽점이 분명하면 `폴리곤`, 픽셀 단위로 칠해야 하면 `브러시`를 선택합니다.
4. 잘못 칠한 부분은 `지우개`로 제거하고 `실행 취소/다시 실행`으로 확인합니다.
5. 왼쪽 저장 라벨 목록에 `폴리곤` 또는 `마스크` 객체가 생겼는지 확인합니다.
6. `라벨 저장`을 누른 뒤 다른 이미지로 이동합니다.
7. 저장한 이미지를 다시 열어 캔버스와 객체 목록이 같은 형태로 복원되는지 확인합니다.

세그멘테이션은 객체탐지 박스를 정답으로 사용하지 않습니다. 앱은 마스크 PNG와 segment JSON을 레시피 정답으로 저장하고, 학습 시 선택한 YOLO-seg 또는 U-Net 어댑터 형식으로 변환합니다.

브러시 크기는 경계보다 지나치게 크지 않게 시작하세요. 넓은 내부는 큰 브러시, 외곽은 작은 브러시로 마무리하면 수정 횟수를 줄일 수 있습니다.

## 5. 이상탐지 OK/NG 판정

![이상탐지 사용 화면](images/workflows-20260722/anomaly-ok-ng-review-1920x1080.png)

현재 이상탐지는 이미지 전체의 `정상(OK)` 또는 `이상(NG)`을 학습하는 2클래스 분류입니다. 결함 위치 박스나 마스크는 필수가 아닙니다.

1. 데이터셋 목적을 `이상 탐지`로 선택합니다.
2. 이미지 루트 하나를 엽니다. 하위 `OK`, `normal`, `NG`, `abnormal` 폴더가 있으면 앱이 폴더명 기준 제안을 보여줍니다.
3. 폴더 구조를 신뢰할 수 있으면 `폴더명 사용`, 확실하지 않으면 `직접 판정`을 선택합니다.
4. `2 라벨링`에서 현재 이미지 전체를 확인합니다.
5. 오른쪽의 `정상(OK) → 다음` 또는 `이상(NG) → 다음`을 누릅니다.
6. 판정 직후 중앙 이미지와 이미지 큐 선택 행이 같은 다음 미판정 이미지로 이동하는지 확인합니다.
7. 잘못 판정했다면 `미판정으로 되돌리기`를 누른 뒤 다시 선택합니다.
8. `다음 미판정`으로 남은 이미지를 확인합니다.

폴더명은 제안일 뿐 정답을 강제하지 않습니다. 이미 수동 저장한 판정은 폴더명 재해석으로 덮어쓰지 않습니다.

결함 위치 정보가 별도로 필요한 프로젝트라면 이상분류와 객체탐지/세그멘테이션 레시피를 목적에 맞게 나눕니다. 이미지 전체 OK/NG와 위치 라벨은 서로 다른 학습 문제입니다.

## 6. 저장 라벨과 AI 후보 구분하기

| 화면 상태 | 의미 |
| --- | --- |
| `저장 필요` | 현재 편집 내용이 아직 파일에 반영되지 않음 |
| `저장됨` | 현재 정답이 저장 폴더에 기록됨 |
| `저장 라벨` | 학습에 사용하는 사람 승인 정답 |
| `AI 후보` | 현재 검사 모델이 제안한 결과. 확정 전에는 정답이 아님 |
| `학습 모델 후보` | 학습은 끝났지만 아직 검사 모델로 채택하지 않은 가중치 |
| `현재 검사 모델` | `현재 검사` 명령이 실제로 사용하는 모델 |

AI 후보가 맞으면 확정하고 틀리면 숨기거나 수정합니다. 후보를 확정한 뒤에도 `라벨 저장`을 눌러야 정답 파일에 반영됩니다.

## 7. 학습 준비

1. `4 학습/모델`로 이동합니다.
2. `데이터`에서 이미지 수, 라벨 수, 클래스 분포, train/valid/test 분할과 오류를 확인합니다.
3. 준비 상태가 실패라면 빨간 안내 문구의 원인부터 해결합니다.
4. `학습/비교`에서 첫 실행은 `빠른 추천 적용`으로 시작합니다.
5. 작업 목적에 맞는 모델 프로필과 시작 가중치를 확인합니다.
6. `시작` 후 epoch 로그, 실패 원인과 결과 경로를 확인합니다.

대표 학습 경로:

| 목적 | 모델 흐름 |
| --- | --- |
| 객체탐지 | YOLOv5, YOLOv8, YOLO11 detect |
| 세그멘테이션 | YOLOv8/YOLO11 segment, U-Net |
| 이상탐지 | YOLOv8 classify |

모델 프로필을 바꾸면 등록된 실행기 기준으로 Python, 프로젝트와 실행 스크립트가 함께 전환됩니다. 일반 사용 중 세 경로를 하나씩 다시 입력하지 않습니다. 자동 탐색이 실패한 첫 연결에서만 실행기 폴더를 지정하고, 화면의 self-test와 학습 가능 상태를 확인합니다.

시작 가중치와 학습 완료 가중치를 구분하세요. 예를 들어 `yolov8n-cls.pt`는 분류 학습을 시작하는 가중치이고, 실행 결과의 `best.pt`는 학습된 후보입니다.

## 8. 후보 모델 검증과 비교

1. 학습 완료 후 새 `best.pt`가 후보 모델로 등록됐는지 확인합니다.
2. 같은 test split, 클래스 목록, 이미지 크기와 임계값을 사용해 기존 모델과 비교합니다.
3. 객체탐지는 precision, recall, 놓침과 과검출을 함께 봅니다.
4. 세그멘테이션은 공통 raster mask 기준 Dice/IoU와 클래스별 오류를 봅니다.
5. 이상탐지는 normal/abnormal confusion과 독립 test 정확도를 봅니다.
6. 조건과 근거가 충분할 때만 후보를 `현재 검사 모델`로 저장합니다.

`test`가 없을 때의 validation 결과는 개발 중 참고값입니다. 생산 모델 교체 근거로 사용하려면 학습과 튜닝에 사용하지 않은 독립 평가 데이터를 준비합니다.

## 9. 현재 검사와 AI 후보 검토

1. 현재 검사 모델이 올바른지 상단 상태와 모델 센터에서 확인합니다.
2. `3 AI 후보`로 이동해 `현재 검사`를 실행합니다.
3. 객체탐지 박스 또는 세그멘테이션 외곽선 후보를 확인합니다.
4. 맞는 후보만 확정하고 틀린 후보는 숨기거나 수정합니다.
5. 확정된 정답을 `라벨 저장`으로 기록합니다.

이상분류 결과는 이미지 단위 normal/abnormal 판정으로 연결됩니다. 이 결과도 사람이 확인한 뒤 정답 상태와 모델 채택 여부를 결정합니다.

## 10. 자주 발생하는 문제

| 증상 | 먼저 확인할 것 |
| --- | --- |
| 이전 라벨이 보임 | 현재 저장 폴더가 이전 데이터셋과 같은지 확인 |
| 하위 NG 폴더가 먼저 보임 | 루트 전체가 한 큐로 로드된 정상 동작인지 파일명과 큐를 확인 |
| OK 이미지를 찾기 어려움 | 이미지 큐를 스크롤하거나 판정/상태 필터 사용 |
| 라벨을 만들었는데 다음 이미지에서 없음 | `라벨 저장` 전 이동했는지 확인 |
| 학습 버튼이 비활성 | 데이터셋 점검과 모델 실행기 self-test 확인 |
| 학습은 끝났지만 검사 모델이 그대로임 | 후보 모델과 현재 검사 모델은 자동 교체되지 않음 |
| 비교 결과가 계산되지 않음 | 데이터 지문, split, 클래스 또는 평가 조건 불일치 확인 |

## 11. 작업 체크리스트

라벨링 전:

- 목적, 이미지 폴더와 저장 폴더 확인
- 클래스 이름과 순서 확인
- 객체탐지 박스, 세그멘테이션 경계 또는 OK/NG 판정 기준 합의

라벨링 중:

- `저장 필요` 상태 확인
- 저장 라벨과 AI 후보 구분
- 이미지 큐의 현재 행과 중앙 이미지 일치 확인
- 수정 필요/검수 완료 상태 기록

학습 후:

- 데이터 분할과 provenance 확인
- 후보 가중치와 시작 가중치 구분
- 동일 조건 비교
- 현재 검사 모델 채택 여부를 명시적으로 결정
