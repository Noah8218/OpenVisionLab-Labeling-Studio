OpenVisionLab Labeling Studio 튜토리얼

처음 실행한 사용자가 객체탐지 데이터셋을 만들고, 박스 라벨을 저장하고, 검출 후보를 검토하는 기본 흐름입니다. 아래 순서대로 진행하면 현재 화면에서 다음에 할 일을 놓치지 않습니다.

1가이드 확인 2이미지 열기 3클래스 등록 4박스 라벨링 5저장과 점검 6검출 후보 검토

이미 검증된 Viewer 성능, ROI 삭제, 브러시/지우개, Candidate Review 기본 동작은 완료 항목입니다. 튜토리얼에서는 사용자가 실제로 눌러야 할 버튼과 판단 순서만 안내합니다.

1

가이드에서 오늘 할 일을 확인합니다

오른쪽 가이드/도구

처음에는 오른쪽 가이드/도구 탭을 봅니다. 현재 단계와 다음 행동이 함께 표시됩니다.

  • 데이터셋 목적이 객체탐지인지 확인합니다.
  • 객체탐지 MVP 완료까지 문구가 다음 작업을 알려줍니다.
  • 단계 카드를 누르면 필요한 화면으로 이동합니다.
2

이미지 큐에서 작업 이미지를 엽니다

왼쪽 이미지 큐

이미지 폴더를 열면 왼쪽 큐에 파일이 표시됩니다. 행을 클릭하면 중앙 캔버스에 바로 열립니다.

  • 샘플 또는 폴더 버튼으로 이미지를 불러옵니다.
  • 다음 버튼은 아직 처리할 이미지로 이동합니다.
  • 후보, 실패, 확정, 스킵, 검출없음 필터로 검토 대상을 줄일 수 있습니다.
3

모델이 배울 클래스를 등록합니다

클래스 탭

박스를 그리기 전에 모델이 배울 이름을 먼저 등록합니다. 예시는 OK, NG, Defect처럼 단순하게 시작합니다.

  • 오른쪽 클래스 탭으로 이동합니다.
  • 클래스 이름을 추가하고 저장 경로를 확인합니다.
  • 클래스 순서는 YOLO 라벨 파일의 class index 기준이 됩니다.
4

박스 라벨을 그리고 저장합니다

라벨링 모드

중앙 캔버스에서 객체를 박스로 표시합니다. 저장하면 같은 이름의 이미지와 라벨 txt가 데이터셋에 기록됩니다.

  • 상단 라벨링 모드에서 박스 도구를 선택합니다.
  • 이미지 밖 검은 영역이 아니라 실제 이미지 위에서 드래그합니다.
  • 오른쪽 객체 탭에서 클래스, 크기, 위치를 확인합니다.
  • 저장을 눌러 현재 라벨을 파일에 반영합니다.
5

데이터셋을 점검하고 학습 준비를 봅니다

YOLO / 가이드

라벨을 저장한 뒤에는 데이터셋 점검으로 이미지, 라벨, 클래스, 분할 상태를 확인합니다.

  • 가이드의 데이터셋 카드 또는 YOLO 화면에서 점검을 실행합니다.
  • 문제가 있으면 클래스 등록, 라벨링, 데이터셋 설정 중 필요한 화면으로 이동합니다.
  • 학습은 점검이 통과된 뒤 시작합니다.
6

검출 후보를 검토합니다

후보 탭

검출을 실행하면 중앙 캔버스와 오른쪽 후보 탭에 검출 후보가 표시됩니다. 후보는 바로 정답이 아니라 검토 대상입니다.

  • 현재 검사로 현재 이미지를 검사합니다.
  • 후보 위치는 선택 후보를 화면 중앙으로 가져옵니다.
  • 기존 라벨은 겹치는 현재 라벨을 선택합니다.
  • 맞으면 확정, 같은 객체라면 스킵하고 다음 후보로 넘어갑니다.
이 문서는 현재 라벨링 워크벤치의 기본 객체탐지 흐름을 기준으로 합니다. 화면 배치가 바뀌면 docs/tutorial/images의 캡처를 함께 갱신합니다.
